# 새 KIND 화면을 API로 추가하기

> 사용자가 *"OO 화면도 api로 만들어줘"* 라고 하면 이 절차대로 추가합니다.
> 예: `https://kind.krx.co.kr/listinvstg/mergeListingCompany.do?method=searchMergeListingCompMain`

핵심: **`krx_kind_data_api/endpoints.py`의 `ENDPOINTS` dict에 한 줄 추가**하면 끝.
대부분은 파서를 새로 짤 필요도 없습니다(`read_html`로 충분).

---

## 0. 한눈에 보기 — 엔드포인트 스펙 필드

| 필드 | 의미 | 기본값 |
|------|------|--------|
| `path` | `.do` 경로 (`listinvstg/mergeListingCompany.do`) | (필수) |
| `http` | `"post"` / `"get"` | `"post"` |
| `send_as` | `"params"`(쿼리) / `"data"`(본문) | `"params"` |
| `encoding` | 응답 디코딩 (`"utf-8"` / `"euc-kr"`) | `"utf-8"` |
| `parser` | `read_html` / `kind_list_with_code` / `today_disclosure` / `corp_list` / 커스텀 | `"read_html"` |
| `parser_kwargs` | 파서 인자 (`{"table_class": "..."}`, `{"table_index": 1}` 등) | `{}` |
| `defaults` | 항상 보낼 파라미터 | `{}` |
| `prepare` | (선택) merged dict를 변형하는 콜러블. 친절한 인자→raw 필드 변환 | 없음 |
| `required` | 호출자가 반드시 채워야 하는 키 | `[]` |
| `screen` | 사람이 읽을 설명 | — |
| `params` | (선택) 호출자/MCP용 파라미터 문서 (`{키: {desc, kind, required, enum, ...}}`) | 없음 |

## 1. 화면의 실제 요청을 알아낸다

브라우저 개발자도구(Network) 또는 기존 코드에서 다음을 확보합니다.
- **URL**: 보통 `...Main`(화면)과 `...Sub`(데이터)이 나뉘어 있음 → **데이터를 주는 `...Sub` 쪽**을 씁니다.
- **HTTP 메서드 / 파라미터 전달 위치**: `requests.post(url, params=...)`면 `send_as="params"`, `requests.post(url, data=...)`면 `send_as="data"`.
- **인코딩**: 한글이 깨지면 `euc-kr`을 의심.
- **테이블 모양**: 그냥 표면 `read_html`, 회사코드가 행의 `onclick`에만 있으면 `kind_list_with_code`.

## 2. `ENDPOINTS`에 한 줄 추가 — `merge_listing` 실제 예시

`searchMergeListingCompMain` 화면의 데이터 호출은 `...Sub`이고, 회사코드는
`onclick="fnDetailView('00126'...)"`에 들어있어 `kind_list_with_code`를 씁니다.

```python
"merge_listing": {
    "path": "listinvstg/mergeListingCompany.do",
    "http": "post",
    "send_as": "params",
    "parser": "kind_list_with_code",
    "parser_kwargs": {"table_class": "list type-00 tmt30"},
    "defaults": {
        "method": "searchMergeListingCompSub",   # ← Main이 아니라 Sub
        "forward": "mergeListingCompany_sub",
        "currentPageSize": "2000",
        "pageIndex": "1",
        "orderMode": "1", "orderStat": "D",
        "marketType": "2",
        "listTypeArrStr": "06|07|",               # 합병상장/재상장
        "secuGrpArrStr": "0|ST|FS|MF|SC|RT|DR|",
        "fromDate": "2000-01-01",
    },
    "required": ["toDate"],
    "screen": "합병·재상장 기업(상장유형 06·07). fromDate~toDate",
},
```

## 3. 바로 사용


## 파라미터를 넘기는 법 (그리고 MCP로 감싸는 법)

`fetch(name, **params)`는 **`spec["defaults"]`에 `**params`를 병합**해서 보냅니다
(`{**defaults, **params}`). 즉 같은 키를 주면 덮어쓰고, 안 주면 기본값이 쓰입니다.
`spec["required"]`에 있는 키가 비면 `KINDFetchError`가 납니다.

> **정리:** 같은 엔드포인트라도 *어떤 필터를 넘겼느냐*에 따라 결과 범위가 달라질 뿐,
> 응답 구조·파서·컬럼은 동일합니다. 필터 없이 부르면 전체, 필터를 주면 그 부분집합입니다.

### `pubofr_prog_com` 파라미터 표

`endpoints.py`의 `"params"` 블록에 기계 판독용으로도 들어 있습니다.

| 파라미터 | 역할 | 필수 | 설명 | 예시 |
|---|---|---|---|---|
| `toDate` | date | ✅ | 조회 종료일(신고서제출일 상한) | `2026-07-02` |
| `fromDate` | date | | 조회 시작일. 생략 시 `2000-01-01` | `2000-01-01` |
| `searchCorpName` | filter | | 회사명(부분일치) **또는** 종목코드. 빈값이면 전체 | `363260` |
| `searchCorpNameTmp` | filter | | `searchCorpName`과 **같은 값**을 넣어야 함(화면이 둘 다 검사) | `363260` |
| `isurCd` | filter | | 발행사 코드 = **종목코드 6자리 중 앞 5자리** (`363260`→`36326`) | `36326` |
| `bzProcsNo` | filter | | 업무처리번호. 특정 공모 건 1개만 조회 | `20200907000341` |
| `repMajAgntComp` | filter | | 대표주선인(증권사)명 | `KB증권` |
| `designAdvserComp` | filter | | 지정자문인(코넥스)명 | |
| `currentPageSize` | paging | | 최대 반환 행 수. **데이터가 아니라 페이지 크기만** 결정. 기본 `2000`≈전량 | `2000` |
| `pageIndex` | paging | | 페이지 번호(1부터) | `1` |
| `orderStat` | sort | | `D`=최신순(내림) / `A`=오름 | `D` |

> `method`/`forward`/`searchMode` 등 나머지 `defaults` 키는 화면 내부 상수라 **건드리지 마세요**.

### 호출 예시

```python
# 전체 공모기업 (필터 없음)
fetch("pubofr_prog_com", toDate="2026-07-02")

# 특정 회사만 (종목코드 363260 = 모비데이즈) — 브라우저 캡처와 동일한 요청
fetch("pubofr_prog_com",
      searchCorpName="363260", searchCorpNameTmp="363260",
      isurCd="36326", toDate="2026-07-02")
```

### MCP 툴로 감싸기 — 이미 구현돼 있음

`endpoint_info(name)["params"]`를 JSON-Schema로 변환해 카탈로그 전체를 MCP 툴로
노출하는 서버가 **[../krx_kind_data_api/mcp_server.py](../krx_kind_data_api/mcp_server.py)** 에 들어 있습니다.

```bash
pip install -e ".[mcp]"
python -m krx_kind_data_api.mcp_server      # stdio 트랜스포트 (= krx-kind-mcp)
```

핵심은 `mcp_server.input_schema(name)` 한 함수입니다(의존성 없는 순수 로직):

```python
from krx_kind_data_api import endpoint_info

def input_schema(name):
    params = endpoint_info(name).get("params", {})
    return {
        "type": "object",
        "properties": {k: {"type": "string", "description": v["desc"]}
                       for k, v in params.items()},
        "required": [k for k, v in params.items() if v.get("required")],
    }
```

서버는 `list_endpoints()`를 돌며 `kind_<엔드포인트>` 툴을 자동 등록하고, 호출되면
`fetch(name, **arguments)` 결과를 `{endpoint, rows, columns, records}` JSON으로 돌려줍니다.

**따라서 새 엔드포인트를 추가할 때 해야 할 일은 `"params"` 블록을 채우는 것뿐입니다.**
그러면 MCP 툴이 자동으로 하나 더 생깁니다(코드 수정 불필요).

### 참고: SPAC 합병상장 — 하나의 화면, 파라미터로 존속/소멸 스위치 (`prepare` 훅 예시)

`mergeListingCompany.do`는 `merge_listing`과 **같은 `.do`**지만, 검색 조건에 따라
전혀 다른 결과가 나옵니다. SPAC(기업인수목적회사) 합병상장만 볼 때는:

- **`listTypeArrStr`** 이 필터를 결정합니다(실측 확인). `06|`=SPAC 존속합병,
  `07|`=SPAC 소멸합병, `06|07|`=둘 다. (같이 오는 `listTypeArr` 리스트는 서버가 무시)
- **`choicTypeArrStr=01|02|03|04|05|`** 를 주면 SPAC 전용 5개 컬럼(액면가·공모가·
  공모금액·주요제품·최초상장주식수)이 추가로 붙습니다. (`choicTypeArr` 리스트는 무시)
- 이 Sub 화면은 `searchCorpName`/`isurCd`/`repMajAgntComp` 같은 회사·주선인 필터를
  **서버단에서 무시**합니다(전체가 그대로 옴). 회사명 검색은 결과 DataFrame에서 거르세요.

존속/소멸을 **친절한 단일 파라미터 `spacType`(존속/소멸/전체)** 로 노출하기 위해
`prepare` 훅을 씁니다. 훅은 merge 직후·required 검증 직전에 실행되어 merged dict를
변형합니다.

```python
_SPAC_TYPE_TO_CODE = {"존속": "06|", "소멸": "07|", "전체": "06|07|"}

def _spac_merge_prepare(p: dict) -> dict:
    spac_type = p.pop("spacType", "전체")          # KIND가 모르는 가상 파라미터 → 제거
    p["listTypeArrStr"] = _SPAC_TYPE_TO_CODE.get(spac_type, spac_type)
    return p

"spac_merge_listing": {
    "path": "listinvstg/mergeListingCompany.do",
    "http": "post", "send_as": "data",
    "parser": "kind_list_with_code",
    "parser_kwargs": {"table_class": "list type-00 tmt30", "columns": [...12개...],
                      "code_column": "회사코드"},
    "prepare": _spac_merge_prepare,                # ← 여기
    "defaults": {..., "choicTypeArrStr": "01|02|03|04|05|", "listTypeArrStr": "06|07|", ...},
    "required": ["toDate"],
    "params": {"spacType": {"kind": "filter", "enum": ["존속","소멸","전체"], ...}, ...},
},
```

```python
fetch("spac_merge_listing", spacType="존속", toDate="2026-07-02")   # SPAC 존속합병만
fetch("spac_merge_listing", spacType="소멸", toDate="2026-07-02")   # SPAC 소멸합병만
fetch("spac_merge_listing", toDate="2026-07-02")                    # 전체(기본)
```

`spacType`은 `params`에 enum으로 문서화돼 있으므로 MCP 툴에도 드롭다운으로 노출됩니다.
raw 필드(`listTypeArrStr`)는 defaults·prepare가 관리하니 MCP엔 노출하지 않습니다.

## 파서가 부족할 때 — 커스텀 파서 추가

표 구조가 특이해 기존 4개 파서로 안 되면 `krx_kind_data_api/parsers.py`에
함수를 만들어 등록합니다. 시그니처는 `(html: str, **kwargs) -> pd.DataFrame`.

```python
# parsers.py 안에서
def my_special_table(html, **_):
    soup = BeautifulSoup(html, "html.parser")
    ...
    return pd.DataFrame(rows)

_PARSERS["my_special_table"] = my_special_table   # 또는 외부에서:
# from krx_kind_data_api import register_parser
# register_parser("my_special_table", my_special_table)
```
그 후 엔드포인트 스펙에서 `"parser": "my_special_table"`로 가리킵니다.

## 자주 만나는 함정

- **`...Main`을 그대로 쓰면 빈 화면**이 옵니다. 데이터는 `...Sub`/`...List` 호출에 있습니다.
- **한글 깨짐** → `"encoding": "euc-kr"` (예: `growth_report`).
- **`params` vs `data`**: 안 되면 반대로 바꿔보세요. KIND 화면마다 다릅니다.
- **회사코드가 비어 있음**: `kind_list_with_code`의 `table_class`가 실제 클래스와
  다르거나, onclick 핸들러 이름이 새로워서일 수 있습니다. `parsers._CODE_RE`에
  핸들러 패턴을 추가하세요.
- **전체 행이 안 옴**: `currentPageSize`를 키우거나 `pageIndex`로 페이지네이션.
